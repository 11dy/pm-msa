       - app/prompts/rag_prompt.py
         · RAG_PROMPT: 시스템 프롬프트 + 컨텍스트 + 질문

         · ROUTER_PROMPT: rag/general 분류
         · GRADER_PROMPT: 문서 관련성 평가 (yes/no)

         · invoke_rag(): 동기 RAG (검색 → 컨텍스트 → LLM)
         · invoke_general(): 동기 일반 응답
         · stream_rag(): 비동기 RAG SSE 스트리밍
         · stream_general(): 비동기 일반 SSE 스트리밍
         · GPT-4o, temperature 0.7

  [✅] 3-3. Adaptive RAG (LangGraph)
       - app/graphs/states/rag_state.py: RAGState TypedDict
       - app/graphs/nodes/router_node.py: GPT-4o-mini로 rag/general 분류
       - app/graphs/nodes/retriever_node.py: Supabase pgvector 문서 검색
       - app/graphs/nodes/grader_node.py: GPT-4o-mini로 문서 관련성 평가, 관련 문서 필터링
       - app/graphs/nodes/generator_node.py: RAG/일반 응답 생성 (GPT-4o)
       - app/graphs/nodes/rewriter_node.py: 쿼리 재작성 + retry_count 증가
       - app/graphs/adaptive_rag.py: LangGraph StateGraph 컴파일

         · grader 후: 관련 문서 있음→generate_rag, 없음+재시도<2→rewrite→retrieve, 없음+재시도≥2→generate_general
         · run_adaptive_rag(): 그래프 실행 + 결과 반환

4. 주요 기술 결정 사항
================================================================================

1. DB 전략
   - WAS용 DB: 로컬 Docker MySQL (pm-auth → dy_db, pm-workflow → pm_workflow)
   - AI/RAG용 DB: Supabase Cloud PostgreSQL + pgvector
   - 배포 환경은 미정, 로컬 Docker로 개발

2. pm-common 제거
   - MSA에서 공통 모듈 추출 불필요
   - 각 서비스가 ApiResponse, BaseEntity 등을 독립 관리

3. WebSocket 프로토콜
   - Spring WebSocket STOMP 사용 (Socket.io 대신)
   - 프론트엔드: @stomp/stompjs

4. 프론트엔드 라이브러리
   - @xyflow/react (구 ReactFlow, v12+에서 리네임)
   - @stomp/stompjs (Socket.io 대신)

5. Kafka 듀얼 리스너
   - INTERNAL (kafka:29092): Docker 내부 통신
   - EXTERNAL (localhost:9092): 호스트 IDE에서 접속

6. 문서 메타데이터 소유권
   - documents 테이블은 pm-workflow(MySQL)가 소유
   - pm-document는 업로드 시 pm-workflow 내부 API 호출로 레코드 생성
   - Gateway에서 업로드(POST)는 pm-document, 조회/삭제(GET/DELETE)는 pm-workflow로 분리

7. 파일 저장소
   - 로컬 파일시스템 (./uploads/<userId>/<uuid>.ext) → 이후 Supabase Storage 전환 가능
   - StorageService 추상화

8. RAG 전략
   - Naive RAG: 단순 검색 → LLM (stream_rag, invoke_rag)
   - Adaptive RAG: LangGraph 기반 (라우팅 → 검색 → 평가 → 생성/재작성)
   - 라우터가 질문을 rag/general로 자동 분류
   - Supabase 미설정 시 일반 LLM 응답으로 fallback

5. 서비스 실행 방법
================================================================================

# 인프라 (Kafka, Redis, Zookeeper, Kafka UI)
docker compose up -d

# MySQL도 함께 (기존 standalone MySQL과 포트 충돌 주의)
docker compose --profile db up -d

# Eureka Server (포트 8761) — 먼저 실행
cd pm-infra && ./gradlew :eureka-server:bootRun

# Gateway (포트 8080)
cd pm-infra && ./gradlew :gateway:bootRun

# pm-auth (포트 8081)
cd pm-auth && DB_PASSWORD=east4317 ./gradlew bootRun

# pm-workflow (포트 8084)
cd pm-workflow && DB_PASSWORD=east4317 ./gradlew bootRun

# pm-document (포트 8082)
cd pm-document && pip install -r requirements.txt && uvicorn app.main:app --port 8082 --reload

# pm-agent (포트 8083) — OPENAI_API_KEY 필요
cd pm-agent && pip install -r requirements.txt && uvicorn app.main:app --port 8083 --reload

# pm-web (포트 3000)
cd pm-web && npm run dev
