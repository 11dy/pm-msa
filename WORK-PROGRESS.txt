4. 주요 기술 결정 사항
================================================================================

1. DB 전략
   - WAS용 DB: 로컬 Docker MySQL (pm-auth → dy_db, pm-workflow → pm_workflow)
   - AI/RAG용 DB: Supabase Cloud PostgreSQL + pgvector
   - 배포 환경은 미정, 로컬 Docker로 개발

2. pm-common 제거
   - MSA에서 공통 모듈 추출 불필요
   - 각 서비스가 ApiResponse, BaseEntity 등을 독립 관리

3. WebSocket 프로토콜
   - Spring WebSocket STOMP 사용 (Socket.io 대신)
   - 프론트엔드: @stomp/stompjs

4. 프론트엔드 라이브러리
   - @xyflow/react (구 ReactFlow, v12+에서 리네임)
   - @stomp/stompjs (Socket.io 대신)

5. Kafka 듀얼 리스너
   - INTERNAL (kafka:29092): Docker 내부 통신
   - EXTERNAL (localhost:9092): 호스트 IDE에서 접속

6. 문서 메타데이터 소유권
   - documents 테이블은 pm-workflow(MySQL)가 소유
   - pm-document는 업로드 시 pm-workflow 내부 API 호출로 레코드 생성
   - Gateway에서 업로드(POST)는 pm-document, 조회/삭제(GET/DELETE)는 pm-workflow로 분리

7. 파일 저장소
   - 로컬 파일시스템 (./uploads/<userId>/<uuid>.ext) → 이후 Supabase Storage 전환 가능
   - StorageService 추상화

5. 서비스 실행 방법
================================================================================

# 인프라 (Kafka, Redis, Zookeeper, Kafka UI)
docker compose up -d

# MySQL도 함께 (기존 standalone MySQL과 포트 충돌 주의)
docker compose --profile db up -d

# pm-auth (포트 8081)
cd pm-auth && DB_PASSWORD=east4317 ./gradlew bootRun

# pm-workflow (포트 8084)
cd pm-workflow && DB_PASSWORD=east4317 ./gradlew bootRun

# pm-document (포트 8082)
cd pm-document && pip install -r requirements.txt && uvicorn app.main:app --port 8082 --reload

# pm-agent (포트 8083)
cd pm-agent && pip install -r requirements.txt && uvicorn app.main:app --port 8083 --reload

# pm-web (포트 3000)
cd pm-web && npm run dev

# Gateway (포트 8080) - Eureka 필요
cd pm-infra && ./gradlew :gateway:bootRun

# Eureka Server (포트 8761)
cd pm-infra && ./gradlew :eureka-server:bootRun
