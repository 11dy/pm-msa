1. 개요
================================================================================

배포 모델: 셀프 호스팅 (Self-Hosted)
- 사용자가 본인 PC에서 Docker Compose로 전체 서비스 실행
- 서버 비용 $0 (사용자 로컬 자원 사용)
- 사용자 부담: OpenAI API 키 비용만

참고 사례: Nextcloud, GitLab CE, Ollama

================================================================================
2. 사전 작업 (릴리즈 전 해결 필요)
================================================================================

[필수] Supabase Cloud 의존성 제거
------------------------------------------------------------------------
현재: pm-agent가 Supabase Cloud PostgreSQL + pgvector 사용
변경: 로컬 Docker PostgreSQL + pgvector로 교체

- docker-compose.yml에 pgvector/pgvector:pg16 컨테이너 추가
- pm-agent의 supabase_store.py → 로컬 PostgreSQL 연결로 변경
- document_chunks 테이블, HNSW 인덱스, match_document_chunks RPC를
  init-pgvector.sql로 자동 생성되도록 작성
- Supabase Storage → 로컬 파일시스템 (이미 구현됨)

[필수] 누락 Dockerfile 작성
------------------------------------------------------------------------
필요한 Dockerfile 목록:
  - pm-workflow/Dockerfile        ← 없음 (Java/Gradle)
  - pm-infra/eureka-server/Dockerfile  ← 확인 필요
  - pm-infra/gateway/Dockerfile        ← 확인 필요
  - pm-auth/Dockerfile                 ← 확인 필요
  - pm-web/Dockerfile                  ← 확인 필요

Java 서비스 Dockerfile 템플릿:
  FROM eclipse-temurin:25-jre
  WORKDIR /app
  COPY build/libs/*.jar app.jar
  ENV JAVA_OPTS="-Xmx256m -Xms128m"
  EXPOSE {port}
  ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -jar app.jar"]

빌드 순서: ./gradlew bootJar → docker build

[필수] OAuth 대안
------------------------------------------------------------------------
현재: Google/Kakao OAuth만 지원 (사용자가 OAuth 앱 등록 필요)
권장: 자체 ID/PW 로그인을 기본 제공, OAuth는 선택 사항으로

[선택] JVM 메모리 최적화
------------------------------------------------------------------------
- 각 Java 서비스 JAVA_OPTS="-Xmx256m -Xms128m" 적용
- docker-compose.yml에서 mem_limit 설정
- 최소 요구 메모리: 4GB, 권장: 8GB

3. 이미지 빌드
================================================================================

3-1. 레지스트리 선택
------------------------------------------------------------------------
옵션 A: Docker Hub (hub.docker.com)
  - 퍼블릭 이미지 무제한 무료
  - 네이밍: {dockerhub-id}/pm-{서비스명}:latest

옵션 B: GitHub Container Registry (ghcr.io)
  - 퍼블릭 무제한 무료, GitHub 계정 연동
  - 네이밍: ghcr.io/{github-id}/pm-{서비스명}:latest

권장: GHCR (GitHub 소스코드와 이미지를 한 곳에서 관리)

3-2. 빌드 명령어
------------------------------------------------------------------------
# Java 서비스 (각 디렉토리에서)
./gradlew bootJar
docker build -t ghcr.io/{id}/pm-auth:latest .
docker build -t ghcr.io/{id}/pm-workflow:latest .
docker build -t ghcr.io/{id}/pm-eureka:latest .
docker build -t ghcr.io/{id}/pm-gateway:latest .

# Python 서비스
docker build -t ghcr.io/{id}/pm-document:latest ./pm-document
docker build -t ghcr.io/{id}/pm-agent:latest ./pm-agent

# Next.js 프론트엔드
docker build -t ghcr.io/{id}/pm-web:latest ./pm-web

3-3. 이미지 Push
------------------------------------------------------------------------
# GHCR 로그인
echo $GITHUB_TOKEN | docker login ghcr.io -u {github-id} --password-stdin

# Push
docker push ghcr.io/{id}/pm-auth:latest
docker push ghcr.io/{id}/pm-workflow:latest
docker push ghcr.io/{id}/pm-eureka:latest
docker push ghcr.io/{id}/pm-gateway:latest
docker push ghcr.io/{id}/pm-document:latest
docker push ghcr.io/{id}/pm-agent:latest
docker push ghcr.io/{id}/pm-web:latest

4. 배포용 파일 구성
================================================================================

사용자에게 제공하는 파일 (GitHub Release 또는 별도 repo):

pm-msa-release/
├── docker-compose.yml      ← image: 참조 (build: 제거)
├── .env.example            ← 사용자가 .env로 복사 후 수정
├── infra/
│   ├── init-db.sql         ← MySQL 초기화
│   └── init-pgvector.sql   ← pgvector 초기화 (테이블, 인덱스, RPC)
└── README.md               ← 설치/실행 가이드

4-1. docker-compose.yml (배포용)
------------------------------------------------------------------------
변경 포인트:
  - build: ./pm-xxx → image: ghcr.io/{id}/pm-xxx:latest
  - Supabase 환경변수 → 로컬 pgvector 연결 정보로 교체
  - 모든 서비스에 mem_limit 설정
  - restart: unless-stopped 추가

예시:
  pm-workflow:
    image: ghcr.io/{id}/pm-workflow:latest
    container_name: pm-workflow
    ports:
      - "8084:8084"
    environment:
      DB_USERNAME: root
      DB_PASSWORD: ${DB_PASSWORD}
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/pm_workflow
      SPRING_KAFKA_BOOTSTRAP_SERVERS: kafka:29092
      EUREKA_CLIENT_SERVICEURL_DEFAULTZONE: http://eureka-server:8761/eureka
      JAVA_OPTS: "-Xmx256m -Xms128m"
    mem_limit: 384m
    restart: unless-stopped
    depends_on:
      mysql:
        condition: service_healthy
      kafka:
        condition: service_started
      eureka-server:
        condition: service_healthy

5. 사용자 설치 가이드 (README.md 내용)
================================================================================

요구사항:
  - Docker Desktop 설치 (Windows/Mac/Linux)
  - 최소 RAM 4GB 여유, 권장 8GB
  - 디스크 여유 공간 5GB 이상

설치 순서:
  1. 파일 다운로드
     git clone https://github.com/{id}/pm-msa-release.git
     cd pm-msa-release

  2. 환경변수 설정
     cp .env.example .env
     # .env 파일 열어서 OPENAI_API_KEY 입력

  3. 실행
     docker compose up -d

  4. 접속
     http://localhost:3000

  5. 종료
     docker compose down

  6. 데이터 삭제 (초기화)
     docker compose down -v

6. CI/CD 자동화 (선택)
================================================================================

GitHub Actions로 이미지 자동 빌드/푸시:

.github/workflows/release.yml:
  on:
    push:
      tags:
        - 'v*'
  jobs:
    build-and-push:
      steps:
        - checkout
        - Java 서비스: gradlew bootJar → docker build → push
        - Python 서비스: docker build → push
        - Next.js: docker build → push
        - GitHub Release 생성 (docker-compose.yml, .env.example 첨부)

태그 기반 배포:
  git tag v1.0.0
  git push origin v1.0.0
  → GitHub Actions가 자동으로 이미지 빌드 + Release 생성

7. 버전 관리
================================================================================

이미지 태깅 전략:
  - latest: 최신 안정 버전
  - v1.0.0: 특정 릴리즈 버전
  - develop: 개발 버전 (선택)

docker-compose.yml에서:
  image: ghcr.io/{id}/pm-workflow:v1.0.0  ← 버전 고정
  또는
  image: ghcr.io/{id}/pm-workflow:latest   ← 항상 최신
